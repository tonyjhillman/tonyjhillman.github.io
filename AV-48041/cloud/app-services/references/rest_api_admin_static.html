Admin REST API

    +
    API reference guide for the Couchbase Capella App Services Admin API.
    The API is regularly updated along with the documentation. In the process, documentation may be built first — so it is possible that a new endpoint listed here may be some hours from reaching the API.

    App Services

    Access

    • HTTP Basic Authentication BasicAuth

    Endpoints

    DatabaseSecurity

    GET /{db}/_role

    Get All Roles

    Description

    This request returns all the roles in the specified database.

    App Services Roles Required :

    • Sync Gateway Architect

    • Sync Gateway Application

    • Sync Gateway Application Read Only

    Parameters
    Path Parameters
    Name Description Required Default Pattern

    db

    Database name

    X

    null
    Return Type

    [List]

    Responses
    Code Message Datatype

    200

    200 OK - Returns the list of roles as an array of strings The message body contains the list of roles in a JSON array. Each element of the array is a string representing the name of a role in the specified database.

    List[[string]]

    401

    401 - Unauthorized - Error validating credentials
    Samples

    DELETE /{db}/_role/{name}

    Delete Specific Role

    Description

    This request deletes the role with the specified name.

    App Services Roles Required :

    • Sync Gateway Architect

    • Sync Gateway Application

    Parameters
    Path Parameters
    Name Description Required Default Pattern

    db

    Database name

    X

    null

    name

    Role name, may contain any combination of the characters [a-z A-Z 0-9 - + . @ %], when creating a role any other characters must be percent encoded, see: https://en.wikipedia.org/wiki/Percent-encoding. When passing a role name in a URL path it must be escaped again using percent encoding for example if a role is created with the name \"0\|59\", the '\|' character must first be percent-encoded resulting in \"0%7C59\". When using the same role name in a URL path it must be percent-encoded a second time resulting in \"0%257C59\"

    X

    null
    Return Type

    -

    Responses
    Code Message Datatype

    200

    200 OK - The role was successfully deleted
    Samples

    UserModel User Configuration Model

    Field Name Required Type Description Format

    name

    X

    [String]

    The user name (the same name used in the URL path).

    The valid characters for a user name are alphanumeric ASCII characters and the underscore character.

    The name property is required in a POST request.

    You don’t need to include it in a PUT request because the user name is specified in the URL.

    password

    [String]

    Password of the user.

    Mandatory, unless allow_empty_password=true.

    admin_channels

    List of [string]

    The channels that the user is able to access.

    admin_roles

    List of [string]

    An array of the roles this user is associated with.

    all_channels

    List of [string]

    Shows the channels the user can access, as granted by the sync function.

    This is a read-only property. Changes to it are ignored.

    email

    [String]

    Email address of the user.

    disabled

    [Boolean]

    This property is usually not included.

    If the value is true, access for the account is disabled and the user will not be able to login.

    roles

    List of [string]

    Shows the roles this user is associated with by the Sync function.

    This is a read-only property. Changes to it are ignored.
